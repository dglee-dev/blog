### 자바스크립트의 등장

1995년, Netscape는 그들의 Navigator 브라우저에 새로운 프로그래밍 언어를 탑재하고 싶어했다. 목적은 프로그래머가 아닌 사람들을 위해 동적인 웹사이트를 만들게끔 돕는 것이었다. Netscape에 고용된 브랜든 아이크가 개발하여 채택되었고, 1995년 9월에 LiveScript 라는 이름으로 먼저 릴리즈 되었지만, 마케팅을 목적으로 당시 잘나가던 Java와 비슷한, **JavaScript**로 그 해 12월 이름이 변경되었다. 초창기의 자바스크립트는 그 기획 의도에 걸맞게 소소한 위젯이나 간단한 유효성 검사(폼 필드 확인) 용도로 사용되었다.

### 모듈 패턴

자바스크립트가 등장한 1995년부터 2000년대 중반까지는 모듈의 개념을 구현한 '모듈 패턴' 은 존재했지만 '모듈 시스템' 은 그다지 대중적이지 않은 존재였다. 모듈 패턴은 기본적으로 전역공간을 오염시키지 않기 위한 노력이었다. 현대의 모듈시스템이 여러 개 파일로 구성된 모듈간의 관계 문제를 해결하는 것과는 다른 차원인 것이다. 모듈 시스템 등장 이후 지칭되는 '모듈' 은 기본적으로 파일 단위를 의미하는 반면, 모듈 패턴만으로 코드를 조직하던 이 당시에 '모듈' 을 지칭하면 파일이 아니라 하나의 스코프를 가진 객체를 의미했다.

모듈 패턴은 주로 IIFE나 Namespace Pattern 등으로 구현되었다.

- Namespace Pattern: 전역 변수 충돌 해결이 목적이었다.

```js
var Company = {};
Company.UI = {};
Company.Util = {};
```

- IIFE: 스코프를 관리하는 것이 목적이었다.


- Module Pattern: 캡슐화가 목적이었다.

```js
var Counter = (function () {
  let counter = 0; // private state

  return {
    increment() {
      // public API
      count++;
    },
  };
})();
```

### AJAX

이처럼 간단한 용도의 언어로 개발되어 사용되던 자바스크립트의 역할이 커지게 된 가장 첫번째 분수령은 AJAX의 등장이다. AJAX는 비동기적으로 데이터를 페칭하고, 가져온 데이터를 바탕으로 웹페이지를 refresh없이 업데이트하는 메커니즘을 의미하는 단어이다. AJAX라는 이름이 탄생하기 전부터 자바스크립트를 이용해 비동기적으로 화면을 업데이트하는 일은 시작되고 발전되어왔는데, 본래 데스크탑 애플리케이션의 유저 경험에 익숙했던 개발자들과 사용자들은 웹페이지에서 하나의 액션을 취할때마다 페이지를 새로고침해야 한다는 지점에 대해 아쉬움이 많았다. 따라서 페이지를 완전히 새롭게 그리지 않고도 화면을 업데이트하고 싶다는 니즈는 자바스크립트가 생겨나기 이전부터 존재했고 클라이언트 측의 상태(UI의 변경, 사용자 입력 등)를 유지시키고 싶다는 생각은 계속되어왔다.

이에 대한 비동기적 업데이트의 초기 시도는 일명 Remote Scripting이라는 방식이었다. iframe을 이용해 페이지와 분리된 컨텍스트에서 추가적인 요청과 응답을 처리했다. 이후 1999년에는 IE5에 탑재되었던 XMLHttp가 XMLHttpRequest라는 이름으로 표준화되었고, 2004년 Gmail과 2005년 Google Maps가 등장하며 마치 데스크탑 애플리케이션과 같은 경험을 제공하는 '웹 애플리케이션' 이라는 개념을 대중화시켰다. 2005년에는 Jesse James Garrett에 의해 AJAX라는 단어가 만들어지며 비동기 웹 기술과 유저 경험은 완전히 주류가 되었다.

그렇게 하나의 도메인 주소가 가리키는 것이 '페이지' 가 아니라 '애플리케이션' 에 가까운 것으로 변화하게 되면서 수 많은 새로운 로직들이 필요하게 되었다. 새로고침되지 않아 보존되는 클라이언트 상태를 관리해야 했고, 새로운 데이터를 화면에 반영하기 위한 DOM 조작, 데이터를 보내거나 받아오기 위한 비동기 흐름 제어, 클라이언트 사이드 라우팅 등을 위한 코드가 자바스크립트 코드베이스에 추가되기 시작한 것이다. 그 결과 클라이언트 측 자바스크립트의 코드량이 폭발적으로 증가하고 여러 사람이 파일단위의 코드를 작성하고 임포트하게 되면서 의존성에 대한 명시적인 관리가 필요해졌고, 자연스럽게 모듈 시스템도 대중화를 이루게 되었다.
